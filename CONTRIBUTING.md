# Contributing to Aira ERP

Thank you for considering contributing to the Aira ERP project! We appreciate your efforts to help improve our software. Please follow these guidelines to make the contribution process seamless:

## How to Contribute

1. **Fork the Repository**: Start by forking the repository to your own GitHub account.

2. **Create a New Branch**: Create a new branch for your work. This helps keep your changes organized.
   
   ```bash
   git checkout -b your-feature-branch
   ```
   
3. **Make Changes**: Make your changes in this new branch.

4. **Commit Your Changes**: Once you are happy with your changes, commit them to your branch.
   
   ```bash
   git commit -m "Add a brief description of your changes"
   ```
   
5. **Push to GitHub**: Push your changes back to your forked repository.
   
   ```bash
   git push origin your-feature-branch
   ```

6. **Open a Pull Request**: Navigate to the original repository and open a pull request. Describe your changes and why they are needed.

## Coding Guidelines
- Follow the existing code style for consistency.
- Write clear, descriptive commit messages.
- Include comments in your code for clarity.

## Reporting Issues
If you encounter any bugs or issues, please report them on the issue tracker, providing as much detail as possible.

Thank you for helping improve Aira ERP! We look forward to your contributions!
